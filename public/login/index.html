La siguiente implementación de Google Sign In no está optimizada.

Este botón llama a una ruta del API endpoint que crea un usuario. Lo correcto sería solo jalar el token y guardarlo en el localStorage. Para crear un usuario con Google, se debería crear otra ruta en /signup y ahí dar ambas opciones a los usuarios: registrarse de forma genérica y registrarse con Google.

Esta aplicación web es estrictamente ilustrativa. Para ver la respuesta del endpoint, abrir la consola del navegador .
